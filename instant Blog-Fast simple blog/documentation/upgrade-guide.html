* Please only follow these steps if you want to upgrade Instant Blog script!
If you downloaded latest version and installing for the first time, you dont need to follow these steps!


* Before you get started, it's a good idea to back up your current website and your database.
This means if there are any issues you can restore your website.


* If you made customization on script or style you need to re-implement those changes on new script.


Upgrading Instant Blog v2.2 to v2.3


REPLACE BELOW FOLDER & FILES WITH NEWLY DOWNLOADED FILES ON YOUR HOSTING
  • instantblog\app\Http\Controllers\InstantController.php
  • instantblog\app\Providers\AuthServiceProvider.php
  • instantblog\database\migrations\



Upgrading Instant Blog v2.1 to v2.2


  1. Go to your hosting file manager
  2. Take copy of these files and folders to your computer :
    - " instantblog/.env" .env file
    - " instantblog/storage/installed" installed file
    - " public/uploads" uploads folder
    - " public/images" images folder

  3. Delete all instantblog script files and folders on your hosting.
  4. Upload newly downloaded files to your hosting as before (on same folder structure ) and extract it.
  5. Change "storage/framework" folder permission to 775.
  6. Change "storage/logs" folder permission to 775.
  7. Change "bootstrap/cache" folder permission to 775.
  8. Upload and replace all copied files (on step 2) to same place on your hosting
  9. Go to yourdomain.com/update click next on update wizard.
  10. When update complated, clear your browser caches.

Upgrading Instant Blog v1.5 to v2.0


  1. Go to your hosting file manager
  2. Take copy of these files and folders to your computer :
    - " instantblog/.env" .env file
    - " instantblog/storage/installed" installed file
    - " public/uploads" uploads folder

  3. Delete all files on your hosting.
  4. Upload newly downloaded files to your hosting as before and extract it.
  5. Open "instant\database\seeds\DatabaseSeeder.php" with text editor and delete these lines and re-upload to hosting ;
    $this->call(UsersTableSeeder::class);
    $this->call(SettingsTableSeeder::class);
    $this->call(PostsTableSeeder::class)
  6. Change "storage/framework" folder permission to 775.
  7. Change "storage/logs" folder permission to 775.
  8. Change "bootstrap/cache" folder permission to 775.
  9. Upload and replace all copied files (on step 2) to same place on your hosting
  10. Go to yourdomain.com/update click next on update wizard.
  11. When update complated, clear your browser caches.



Upgrading Instant Blog v1.4 to v1.5


UPLOAD OR REPLACE BELOW FILES WITH NEWLY DOWNLOADED FILES ON YOUR HOSTING
  • instantblog\config\purifier.php
  • instantblog\resources\lang\en\messages.php
  • instantblog\resources\views\auth\login.blade.php
  • instantblog\resources\views\auth\passwords\email.blade.php
  • instantblog\resources\views\auth\passwords\reset.blade.php
  • instantblog\config\app.php
  • Edit "instantblog\.env" file with your text editor and replace these lines:
    MAIL_DRIVER=smtp
    MAIL_HOST=mailtrap.io
    MAIL_PORT=2525
    MAIL_USERNAME=null
    MAIL_PASSWORD=null
    MAIL_ENCRYPTION=null

    with:

    APP_NAME=YourSiteName
    MAIL_DRIVER=smtp
    MAIL_HOST=mailtrap.io
    MAIL_PORT=2525
    MAIL_USERNAME=null
    MAIL_PASSWORD=null
    MAIL_ENCRYPTION=null
    MAIL_FROM_ADDRESS=from@yoursite.com
    MAIL_FROM_NAME=ExampleName
     and fill informations, save and re-upload to hosting. You can check readme for details.

Upgrading Instant Blog v1.2 to v1.3


* On this video tutorial we used localhost but progress will be same for hosting too.
Just pay attention on folder/file names and actions.


  1. Go to your hosting file manager
  2. Download the ".env" file inside of "instantblog" folder to your desktop.
  3. Download the "installed" file inside of "instantblog/storage" folder to your desktop.
  4. Delete all files inside of "instantblog" folder on your hosting.
  5. Upload newly downloaded "instantblog.zip" to inside of "instantblog" folder and extract it.
  6. Change "storage/framework" folder permission to 775.
  7. Change "storage/logs" folder permission to 775.
  8. Change "bootstrap/cache" folder permission to 775.
  9. Edit ".env" file with your text editor add "PINTEREST_ACCESS_TOKEN=XXXXXXXXXXXXXXXXXXXXXXXX". and save it. You can check readme for creating pinterest api.
  10. Upload and replace ".env" file from your desktop (downloaded on step 2) to inside of "instantblog" folder.
  11. Upload "installed" file from your desktop (downloaded on step 3) to inside of "instantblog/storage" folder.
  12. Upload and replace newly downloaded "form.js" file to "public\js" folder on our hosting.
  13. Clear your browser caches.
